<h1>_dl vers 5 - 030993</h1>

<h1>&#8212;- technical details : &#8212;&#8212;&#8212;&#8212;</h1>

<h1> </h1>

<h1>This ircd.conf uses m4 to lighten the configuration work.</h1>

<h1>You need to have #define M4_PREPROC in the server config.h</h1>

<h1>for this to work, and have a file called &#8220;ircd.m4&#8221; in the IRCD directory.</h1>

<h1>ircd.m4 is created by &#8220;make install&#8221;</h1>

<h1>Currently I don&#8217;t use the macros in ircd.m4, so it can be empty :)</h1>

<h1>it just has to be there&#8230; so that m4 doesn&#8217;t fail miserably.</h1>

<h1>&#8220;touch ircd.m4&#8221; >:-)</h1>

<p>#</p>

<h1>The defaults classes for the CLIENTS and the SERVERS should be quite okay,</h1>

<h1>basically you only need to setup the SERVER INFO at the beginning and</h1>

<h1>insert uplinks and downlinks at the bottom.</h1>

<p>#</p>

<h1>Remember this is only some <em>basic</em> settings to keep things <em>simple</em>.</h1>

<h1>You don&#8217;t get the full features in there, but it should be more than</h1>

<h1>enough for normal use.</h1>

<h1>You should check doc/example.conf and doc/INSTALL</h1>

<h1>for more detailed explanations of all available options.</h1>

<p>#</p>

<h1>The ircd.conf is parsed and stored in such a way that it&#8217;s as if</h1>

<h1>it was read <em>backwards</em>, so if you want for instance to have a C/N</h1>

<h1>pair tried first, it should be located <em>LAST</em> in the ircd.conf.</h1>

<p>#</p>

<h1>To get to a traditionnal ircd.conf, all you need to type is:</h1>

<p>#</p>

<h1>m4 ircd.conf | grep -v ^# | uniq</h1>

<p>#</p>

<h1>Also remember to run &#8220;ircd/chkconf&#8221; and see if there is no problem!!!</h1>

<p>#</p>

<h1>&#8212;&#8212;&#8212;- Macros definitions&#8230; don&#8217;t edit unless you&#8217;re really sure &#8212;&#8212;&#8212;</h1>

<h1>&#8212;&#8212;&#8212;- of what you are doing ! Skip that part&#8230; search %%%%      &#8212;&#8212;&#8212;</h1>

<h1>PING_SERVER : server ping frequency in seconds</h1>

<h1>PING_CLIENT : client ping frequency in seconds</h1>

<p>define(PING<em>SERVER,120)
define(PING</em>CLIENT,300)</p>

<h1>&#8212;- Client Class.</h1>

<h1>CLIENT_MAX   : is the maximum number of clients you want connected.</h1>

<h1>CLIENT_CLASS : we&#8217;ll put clients in class 1, no change needed.</h1>

<h1>CLIENT_SENDQ : limit in SendQ for clients, in bytes.</h1>

<p>define(CLIENT<em>MAX,100)
define(CLIENT</em>CLASS,1)
define(CLIENT_SENDQ,40000)</p>

<h1>&#8212;- Server Class.</h1>

<h1>I usually define those classes for the servers, they may or may not</h1>

<h1>be appropriate for you.</h1>

<h1>A basic installation may not need the AUTOCON class, you can fairly</h1>

<h1>do with DOWNLINK and UPLINK.</h1>

<p>#</p>

<h1>Class explanations:</h1>

<h1>DOWNLINK : many downlinks, no autoconnect.</h1>

<h1>UPLINK   : max 1 server connected, autoconnect.</h1>

<h1>AUTOCON  : no max, autoconnect.</h1>

<h1>Note that uplinks you don&#8217;t connect to normally go into the DOWNLINK class.</h1>

<h1>DOWNLINK_CLASS    : we&#8217;ll put downlinks in class 5, no change needed.</h1>

<h1>AUTOCON_CLASS     : we&#8217;ll put autoconnect in class 8, no change needed.</h1>

<h1>UPLINK_CLASS      : we&#8217;ll put uplinks in class 9, no change needed.</h1>

<p>define(DOWNLINK<em>CLASS,5)
define(AUTOCON</em>CLASS,8)
define(UPLINK_CLASS,9)</p>

<h1>SERVER_SENDQ      : limit in SendQ for servers, in bytes.</h1>

<h1>SERVER_MAXCONNECT : limit in the number of servers connected, per class.</h1>

<h1>UPLINK_CONNECT    : seconds for autoconnecting to the uplink.</h1>

<h1>AUTOCON_CONNECT   : seconds for autoconnecting to the servers.</h1>

<p>define(SERVER<em>SENDQ,400000)
define(SERVER</em>MAXCONNECT,20)
define(UPLINK<em>CONNECT,100)
define(AUTOCON</em>CONNECT,300)

Y:AUTOCON<em>CLASS:PING</em>SERVER:AUTOCON<em>CONNECT:SERVER</em>MAXCONNECT:SERVER<em>SENDQ
Y:UPLINK</em>CLASS:PING<em>SERVER:UPLINK</em>CONNECT:1:SERVER_SENDQ</p>

<h1>you can skip to %%%% for simpler definition&#8230;</h1>

<p>#</p>

<h1>Syntax for the definition of server links is:</h1>

<h1>servername        : user@host of the remote server</h1>

<h1>masked_servername : under which mask will the remote server will be known</h1>

<h1>serverport        : remote server port number</h1>

<h1>passwd            : password of the link</h1>

<h1>stripcount        : how many parts of <em>MY</em> server name to strip.</h1>

<h1>for instance, if my servername is a.b.c.d.edu</h1>

<h1>and my stripcount 2, then the server will be seen on</h1>

<h1>the network as *.c.d.edu</h1>

<p>#</p>

<h1>SERVER_DOWNLINK(Servername,Passwd [,MaskedServername [,Stripcount]] )</h1>

<h1>SERVER_AUTOCON(Servername,Passwd</h1>

<h1>[,MaskedServername [,Serverport [,Stripcount]] )</h1>

<h1>SERVER_UPLINK(Servername,Passwd</h1>

<h1>[,MaskedServername [,Serverport [,Stripcount]] )</h1>

<p>#</p>

<h1>If no MaskedServername, it&#8217;s assumed to be the same as Servername.</h1>

<h1>If Serverport is not specified a default value of 7000 is assumed.</h1>

<h1>If Stripcount is not there, your server won&#8217;t be hostmasked.</h1>

<p>#</p>

<h1>Remember, backup uplinks you don&#8217;t connect to normally (no autoconnect)</h1>

<h1>go into the DOWNLINK list! sorry if the name is a little misguiding&#8230;</h1>

<p>#</p>

<h1>MaskedServername is the <em>EXACT</em> name of the server as masked on the IRCnet.</h1>

<h1>ServerForbidden or ServerAllowed can have wildcards.</h1>

<h1>For SERVER_LEAF :</h1>

<h1>the link to that server will be dropped if it tries to introduce</h1>

<h1>- a server matching the ServerForbidden mask</h1>

<h1>- more servers than the specified count</h1>

<h1>For SERVER_HUB :</h1>

<h1>the link will be dropped if the server tries to introduce</h1>

<h1>- a server for which a matching ServerAllowed mask does not exist.</h1>

<p>#</p>

<h1>SERVER_LEAF(MaskedServername [,ServerForbidden [,count]] )</h1>

<h1>SERVER_HUB(MaskedServername [,ServerAllowed] )</h1>

<p>#</p>

<h1>.undernet.org is added automagically, if you need</h1>

<h1>non &#8216;UnderNet.org&#8217; versions use _STD macros&#8230;</h1>

<p>#</p>

<h1>than 7000])</h1>

<h1>for all your uplinks, don&#8217;t add .UnderNet.org, it is added automatically</h1>

<h1>do the same with your downlinks if you have&#8230;(using SERVER_DOWNLINK())</h1>

<h1>(H lines for uplinks are added too)</h1>
