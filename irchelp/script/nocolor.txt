                                <p>if (0){
     Script: nocolor, http://www.best.com/~linux/ircII/nocolor
     Author: lzh@linux.vip.best.com, lzh@Undernet#ircII
Description: ircII script to remove mIRC color codes
              as specified in http://www.best.com/~linux/ircII/mIRCcolorSpec
    Version: $Id: nocolor,v 2.4 1997/02/04 12:05:20 lzh Exp $
Adtl Kywrds: colour stripper filter out cstrip</p>

<ol>
<li>save to nocolor</li>
<li>/load nocolor</li>
<li>no more color codes</li>
</ol>

<p>This version is a lot faster than the 1.x versions because it operates
on as many characters as it can at a time.</p>

<p>Installation</p>

<p>Save this to a file named nocolor if you don&#8217;t already have it
there. The name is important as it is used in the /nocolor command. If
you change the name you will need to change it in the nocolor alias.</p>

<p>at the unix command prompt do:</p>

<p>cd              # without any arguments should return you to your home dir
 mkdir .irc      # makes the .irc dir  (ircII&#8217;s default user script dir)
 mv nocolor .irc # moves nocolor to .irc</p>

<p>echo ~          # shows what ~ means as used below.</p>

<p>shows (for me):</p>

<p>/home/lzh</p>

<p>irc             # start ircII (use other command if appropriate)</p>

<p>In ircII:</p>

<p>/set load       # note(ircII allows you to abbreviate many things)</p>

<p>ircII displays (for me):</p>

<p>*** Current value of LOAD_PATH is ~/.irc:/usr/lib/irc/script</p>

<p>If ~/.irc isn&#8217;t in LOAD_PATH and you want to add it:</p>

<p>/eval set load<em>path ~/.irc:$load</em>path</p>

<p>If that was needed you most likely would also want to add it to your
~/.ircrc, as noted below.</p>

<p>/which nocolor</p>

<p>ircII displays (for me):</p>

<p>*** /home/lzh/.irc/nocolor</p>

<p>/load nocolor</p>

<p>ircII displays:</p>

<p>*** Viewing Color Codes turned off
 *** You can use /nocolor to toggle viewing Color Codes</p>

<p>voila! no more mIRC colors. :)</p>

<p>/nocolor</p>

<p>ircII displays:</p>

<p>*** Viewing C1,4ColorC Codes turned on
 *** Viewing Color Codes turned on</p>

<p>With color viewing on you&#8217;ll see both the line containing the codes
followed by the line with them removed.</p>

<p>/nocolor</p>

<p>ircII displays:</p>

<p>*** Viewing Color Codes turned off</p>

<p>If you would like nocolor loaded whenever you run ircII you can add:</p>

<p>load nocolor</p>

<p>to the .ircrc file in your home directory. If one doesn&#8217;t exist of
course you can create it with a text editor.</p>

<hr />

<p>In the following ^C or ^<letter> means a single control character. If
you would like to produce a ^C, /bind ^c self_insert should work. Note
in the /bind command an actual ^ is typed.</p>

<p>mIRC users apparently type ^K to insert the ^C and then enter the
foreground and background numbers or just ^C to turn off a color
change. To ircII users this appears as an inverse C followed by 1 or 2
digits and possibly a comma and 1 or 2 more digits. Beautifully
intermixed with the other characters to look like an awful mess.</p>

<p>Example (you usually won&#8217;t see something this bad):</p>

<p>C11,11PCC11,11rC11,11eCC11,11tCC11,11tCC11,11yC C11,11HCC11,11CuCC11,11hC?
        P       r      e       t       t       y        H        u       h ?</p>

<p>Note the 1&#8217;s are just there to show where a color value would go.</p>

<p>www servers negotiate with the client to determine what the client
supports, why not irc servers? In that scheme color codes would only
be seen if requested, leaving the existing clients unchanged.</p>

<p>If you need or want to put a real Control C in [^C],</p>
